Demo 3: Multi-Chunk Formatting Styles
============================================================
Compare three formatting styles for the same 4 chunks.

  Style: 'plain' (723 chars)
    RAG (Retrieval-Augmented Generation) combines retrieval with ...
    ... (7 total lines)

  Style: 'labeled' (848 chars)
    --- Source: rag_intro.txt ---
    RAG (Retrieval-Augmented Generation) combines retrieval with ...
    ... (11 total lines)

  Style: 'numbered' (892 chars)
    [1] (Source: rag_intro.txt, Score: 0.92)
    RAG (Retrieval-Augmented Generation) combines retrieval with ...
    ... (11 total lines)

  Trade-offs:
  • Plain: shortest, no source tracking, hardest to evaluate
  • Labeled: clear sources, moderate length, easy to read
  • Numbered: best for citation, LLM can reference [1], [2], etc.

Demo 4: Insufficient Context Handling
============================================================

  Case 1: No context retrieved
  --------------------------------------------------------
  | You are a helpful assistant.
  | 
  | No relevant context was found in the knowledge base for this question.
  | Respond by acknowledging that you cannot find the information ...
  | 
  | Question: What is the capital of France?
  | 
  | Answer:
  --------------------------------------------------------

  Case 2: Irrelevant context (scores < 0.3)
  --------------------------------------------------------
  | [1] (Source: devops_guide.txt, Score: 0.25)
  | Kubernetes orchestrates container deployment across clusters.
  | 
  | [2] (Source: python_basics.txt, Score: 0.18)
  | Python virtual environments isolate project dependencies.
  | ...
  --------------------------------------------------------

  Key lesson: The grounding instruction tells the LLM to say
  "I cannot find this" rather than hallucinating an answer.
  In production, also filter chunks by score threshold before prompting.

Demo 5: Context Window Budget Analysis
============================================================

  Question: "What is RAG and how does it work?"

   Chunks   Prompt Chars   ~Tokens   % of 4k   % of 8k
  -------  -------------  --------  --------  --------
        1            579       144      3.5%      1.8%
        2            805       201      4.9%      2.5%
        3           1031       257      6.3%      3.1%
        4           1249       312      7.6%      3.8%

  Budget allocation for a 4096-token model:
    System + instructions:  ~100 tokens
    Question:               ~20 tokens
    Reserved for response:  ~500 tokens
    Available for context:  ~3476 tokens (~13,900 chars)
    At ~200 chars/chunk:    ~69 chunks max

  ⚠  With small context windows (4k), you must either:
     • Limit top-k retrieval (fewer but more relevant chunks)
     • Use shorter chunks (sacrifice context for precision)
     • Summarize chunks before inserting into prompt
