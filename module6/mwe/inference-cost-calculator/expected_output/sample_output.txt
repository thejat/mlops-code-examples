--- The Master Cost Formula ---

  Cost per request ($) =
    output_tokens × time_per_token_sec × (gpu_cost_per_hour / 3600)
    ÷ batch_size
    × (1 - cache_hit_rate)

  Dimensional analysis:
    tokens × sec/token × $/hour × hour/3600sec ÷ batch × (1-hit_rate)
    = $ per request

--- Baseline: Unoptimized Serving ---

  Configuration:
    Model: 7B FP16 on A100 80GB
    Throughput: 100 tokens/sec
    GPU cost: $4.50/hour
    Batch size: 1 (no batching)
    Cache hit rate: 0%

  Cost per request (200 tokens): $0.0025
  Cost per million tokens:       $12.50

========================================================================
KEY TAKEAWAYS
========================================================================

  Every variable in the cost equation is an optimization lever:

  Variable           Impact      How to Optimize
  ------------------------------------------------------------
  output_tokens      Linear      Limit max_tokens, shorter prompts
  time_per_token     Linear      Quantization, faster GPU
  gpu_cost_per_hour  Linear      Reserved instances, spot pricing
  batch_size         Inverse     Continuous batching
  cache_hit_rate     Inverse     Better caching, semantic cache

  Decision guide (for GPT-4 class API pricing):
  - < 350M tokens/month → Use APIs (self-hosting overhead exceeds savings)
  - 350M–1B tokens/month → Evaluate self-hosting (break-even zone)
  - > 1B tokens/month   → Self-host, optimize aggressively
  Note: With cheap APIs (GPT-4o mini), thresholds shift much higher.
